# Klinische registers

De **klinische registers** structureren de opvolging van risico's en geriatrische
evaluaties. Elk register bundelt de evaluaties van eenzelfde thema, in de tijd,
voor elke bewoner.

## De beschikbare registers

| Register | Waarvoor het dient |
|---|---|
| **Vallen** | Valincidenten registreren en analyseren |
| **Doorligwonden (Braden)** | Risico op doorligwonden via de Braden-schaal |
| **Wonden** | Opvolging van wonden en hun genezing |
| **Pijn** | Evaluatie en opvolging van pijn |
| **Fixatie** | Traceerbaarheid van fixatiemaatregelen |
| **Voeding (MNA)** | Screening van ondervoeding (Mini Nutritional Assessment) |
| **Continentie** | Opvolging van de continentie |
| **Mobiliteit (Tinetti)** | Evenwicht en valrisico |
| **Cognitie (MMSE)** | Cognitieve functies |

## Hoe het werkt

1. Open het gewenste register vanuit de toepassing Zorg → **Registers**.
2. **Nieuwe evaluatie** voor de betrokken bewoner: voer de items in; Resthome
   berekent de **score** en het risiconiveau.
3. Het register **volgt de evolutie**: elke nieuwe evaluatie wordt aan de
   geschiedenis toegevoegd.
4. Een open register kan **afgesloten / stopgezet** worden, en daarna **heropend**
   indien nodig.

!!! note "Verbonden met facturatie en eHealth"
    Sommige evaluaties hebben elders impact: de **MNA** voedt de nutritionele
    opvolging (aan [Maaltijden](../repas/index.md)-zijde), en de **Katz** bepaalt
    het forfait (zie [De Katz-evaluatie](../residents/katz.md)).

!!! tip "Het juiste register op het juiste moment"
    Open een register zodra een risico wordt vastgesteld (na een val, bij het
    verschijnen van een wond…): de opgebouwde geschiedenis vergemakkelijkt de
    overdrachten en de controles.
