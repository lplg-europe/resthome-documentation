# Registres cliniques

Les **registres cliniques** structurent le suivi des risques et des évaluations
gériatriques. Chaque registre regroupe les évaluations d'un même thème, dans le
temps, pour chaque résident.

## Les registres disponibles

| Registre | À quoi il sert |
|---|---|
| **Chutes** | Consigner et analyser les chutes |
| **Escarres (Braden)** | Risque d'escarres via l'échelle de Braden |
| **Plaies** | Suivi des plaies et de leur cicatrisation |
| **Douleur** | Évaluation et suivi de la douleur |
| **Contention** | Traçabilité des mesures de contention |
| **Nutrition (MNA)** | Dépistage de la dénutrition (Mini Nutritional Assessment) |
| **Continence** | Suivi de la continence |
| **Mobilité (Tinetti)** | Équilibre et risque de chute |
| **Cognition (MMSE)** | Fonctions cognitives |

## Comment ça marche

1. Ouvrez le registre voulu depuis l'application Soins → **Registres**.
2. **Nouvelle évaluation** pour le résident concerné : saisissez les items ;
   Resthome calcule le **score** et le niveau de risque.
3. Le registre **suit l'évolution** : chaque nouvelle évaluation s'ajoute à
   l'historique.
4. Un registre ouvert peut être **clôturé / arrêté**, puis **rouvert** si besoin.

!!! note "Relié à la facturation et à l'eHealth"
    Certaines évaluations ont un impact ailleurs : le **MNA** alimente le suivi
    nutritionnel (côté [Repas](../repas/index.md)), et le **Katz** conditionne le
    forfait (voir [L'évaluation Katz](../residents/katz.md)).

!!! tip "Le bon registre au bon moment"
    Ouvrez un registre dès qu'un risque est identifié (après une chute, à
    l'apparition d'une plaie…) : l'historique constitué facilite les transmissions
    et les contrôles.
